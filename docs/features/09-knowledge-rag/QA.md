# QA — 09-knowledge-rag

## 1. Risk assessment
| Risk | Likelihood | Impact | Test focus |
|---|---|---|---|
| Answer invents facts not in the docs | M | **H** (DNA) | Extractive answer ⊂ source text |
| Wrong passage retrieved | M | M | TF-IDF ranking unit |
| PDF text not extracted | M | M | pdf.js round-trip integration |
| Empty corpus/query crashes | L | M | Graceful messages |
| pdfjs-dist leaks past the adapter | L | M | Boundary check (CI) |

## 2. Test approach by level
- **Unit (domain)**: `chunkText` splits; `retrieve` ranks the matching chunk first + respects k; `composeAnswer` returns answer text drawn from the source + a citation to the right doc; empty cases.
- **Integration (backend)**: the pdf.js adapter extracts known text from a PDF generated by our pdf-lib adapter.
- **Integration (frontend)**: Knowledge pane — ingest text → ask → an answer + a citation chip appear; the answer references the ingested content.
- **E2E**: Knowledge → add a document → ask a question → answer + citation visible.
